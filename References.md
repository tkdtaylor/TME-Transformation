# References

This file records the external sources that inform the frameworks and identifies which references should be rechecked regularly. Regulatory and standards references change over time; do not treat this list as legal advice.

**Last reviewed:** 2026-05-15

## Review Cadence

| Reference Type | Suggested Cadence | Why |
|---|---|---|
| Regulation and enforcement guidance | Quarterly | Deadlines, guidance, and implementation timelines can change |
| Standards and security taxonomies | Semi-annually | New versions and mappings may appear |
| Cloud and adoption frameworks | Annually | Concepts are stable, but official guidance evolves |
| Metrics and research references | Annually | Benchmarks and terminology shift over time |

## Maintenance Notes

- When a source changes materially, update the affected framework section and record the change in [CHANGELOG.md](CHANGELOG.md).
- For regulatory references, prefer official government, regulator, or standards-body sources over commentary.
- If a deadline or obligation is still in flux, describe it as staged guidance and include a "last reviewed" date rather than presenting it as permanently settled.
